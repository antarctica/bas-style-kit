How to use

For performance reasons, all icons require a base class and individual icon class. To use, place the following code just about anywhere. Be sure to leave a space between the icon and text for proper padding.

The base class will depend on the which icon font the icon comes from:

  • Font Awesome fa fa-*
  • Map Glyphs mg map-*
  • Devicons devicons devicons-*

Don't mix with other components

Icon classes cannot be directly combined with other components. They should not be used along with other classes on the same element. Instead, add a nested <span> and apply the icon classes to the <span>.

Only for use on empty elements

Icon classes should only be used on elements that contain no text content and have no child elements.

Changing icon font locations

All of the icon fonts used in the BAS Style Kit assume their font files will be located in the ../fonts/ directory, relative to the compiled CSS files. We override these default locations to use namespaced directories (e.g. for Font Awesome the default is changed to ../fonts/font-awesome). You can override both of these defaults in one of two ways depending on how you are using the Style Kit.

  • If using Less files, change the relevant variables in less/variables.less.
  • If using the pre-compiled CSS styles, change the url() paths as needed.

Availability of Glyphicons

Bootstrap, the underlying framework used by the BAS Style Kit, includes Glyphicons halflings by default. The use of this icon font is NOT supported by the BAS Style Kit and SHOULD NOT be used. Font Awesome provides compatibility with glyphicons if needed.

Annoyingly because we don't modify Boostrap itself we cannot remove the references to the Glyphicon font files, though they are not used. This leads to missing resource errors. Though this will only be visible in developer tools, the Style Kit still includes these font files. This isn't ideal since they will never be used, however providing the CDN is used its likely these resources will already be cached.

Accessible icons

Modern versions of assistive technologies will announce CSS generated content, as well as specific Unicode characters. To avoid unintended and confusing output in screen readers (particularly when icons are used purely for decoration), we hide them with the aria-hidden="true" attribute.

If you're using an icon to convey meaning (rather than only as a decorative element), ensure that this meaning is also conveyed to assistive technologies â€“ for instance, include additional content, visually hidden with the .sr-only class.

If you're creating controls with no other text (such as a <button> that only contains an icon), you should always provide alternative content to identify the purpose of the control, so that it will make sense to users of assistive technologies. In this case, you could add an aria-label attribute on the control itself.

Stacked icons

These examples work with the font-awesome icon web-fonts only.

To stack multiple icons, use the fa-stack class on the parent, the fa-stack-1x for the regularly sized icon, and fa-stack-2x for the larger icon. fa-inverse can be used as an alternative icon colour. You can even throw larger icon classes on the parent to get further control of sizing.
