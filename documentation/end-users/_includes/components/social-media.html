Social media

The social media component is used provide a consistent way to symbolise the social media services BAS uses.

See the BAS Style Guide for more information on these services, including current URLs.

This component is unique to the BAS Style Kit.

Supported services

The following services are currently supported Twitter (.twitter), Facebook (.facebook), Youtube (.youtube) and RSS (.rss).
If you need support for another service, or you've used another service already please let us know or submit a pull request.

Use of branded assets

Where possible for any service used by the BAS Style Kit we will use officially provided assets, such as logos.

All brand assets are usually trademarks of their respective owners and therefore subject to licensing restrictions.
You must ensure that where you use a service's assets through the BAS Style Kit that you comply with these restrictions.

We cannot provide general guidance on acceptable usage here, as it is context specific, however if you need any help please let us know.
Finally, the use of these trademarks does not indicate endorsement of the trademark holder by BAS, nor vice versa.

Static symbols

Used to refer to a particular service. Wherever possible we use the Font Awesome icon for the relevant service, coloured with the service's primary brand colour.

To use include the class of the service and .social-media-bsk-static as a base class as a wrapper around an i.fa.fa-3x element containing the relevant service icon.

Linked symbols

Builds on the static symbol and adds styles for link hover etc. Use .social-media-bsk-link as a base class, instead of .social-media-bsk-static.<

List of symbols

If using multiple symbols styles are provided to arrange these in a nice list by wrapping li's containing a static or linked symbol in a ul.list-inline list with the additional .social-media-bsk-list class as well.

Supported components

Social media lists are currently only supported when used within the Footer component, and then only when using the BAS Style Kit variants.
If you need support for another component, or you've done testing yourself using another component please let us know or submit a pull request.

Do not use .list-inline and .fa-ui together

These styles are incompatible as the Font Awesome styles assume the use of a vertical list.
In practical terms this doesn't make a difference as in the inline list styles present list items with icons in an acceptable way.
