Sidebar

Sidebars are used to provide accompanying content side by side with main content

This component is unique to the BAS Style Kit, though loosely based on the well component.

Default sidebar

You should always use a sidebar variant

As with buttons and other components you should always use a sidebar variant, such as .sidebar-bsk-default or .sidebar-bsk-inverse not just the base class.

Additional styles needed if not using a heading

Add .sidebar-bsk-no-heading if a sidebar doesn't start with a heading (h2, h3 or h4) to add some extra padding.
This makes the top edge of the sidebar box meet the top of the main content heading and look right.

Sidebar sizes

Add .sidebar-bsk-lg to use a more padding within a sidebar, use the grid system to make sidebars wider or narrower overall.

Nested sidebars

Its possible to nest a sidebar within another sidebar. This is only supported for one level deep and for sidebars within sidebars.
However, it should be relatively easy to make work at other depths or elements, please submit a pull-request if you do for this so we can make this component better for everyone.
