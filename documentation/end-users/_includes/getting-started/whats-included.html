What's included

Customised Bootstrap CSS required

The BAS Style Kit depends on a customised version of Bootstrap's CSS to override some key settings (such as the default border-radius).
You must include these custom Bootstrap styles before the Style Kit's main CSS. See the Basic template for an example.

jQuery and Bootstrap JS required

The BAS Style Kit depends on Bootstrap's JavaScript plugins, which depend on jQuery, for providing component interactivity.
Therefore you must include jQuery before Bootstrap, see the Basic template for an example.

Source styles

Once cloned from the project repository, you'll see something like this:

This is not a full file/directory tree as that would be impractical and not very useful. Instead only high level or important files/directories are listed, with additional guidance given after.

{% highlight bash %} bas-style-kit/ ├── dist │   ├── ... ├── documentation │   ├── developers │   │   └── index.md │   ├── end-users │   │   ├── ... │   └── project-management │   └── index.md ├── fonts │   └── mapglyphs │   ├── MapGlyphs.eot │   ├── MapGlyphs.ttf │   └── MapGlyphs.woff ├── js ├── less │   ├── ... ├── node_modules │   ├── ... ├── provisioning │   ├── ... ├── site │   ├── ... ├── _config.yml ├── ansible.cfg ├── ansible-roles-external.yml ├── ansible-roles-internal.yml ├── CONTRIBUTING ├── gulpfile.js ├── LICENSE ├── LICENSE-bootstrap ├── package.json ├── README.md ├── terraform.tf ├── terraform.tfstate ├── terraform.tfstate.backup ├── terraform.tfvars ├── terraform_variables.tf └── Vagrantfile {% endhighlight %}
  • /dist contains the same files found in the Compiled styles
  • /documentation contains this documentation (as a Jekyll website) and additional, text only, documentation for development and project management purposes
  • /fonts contains the Map Glyph icon font as this isn't available as a package through NPM
  • /js is an intentionally empty directory. In the future this will be used for custom JavaScript plugins
  • /less contains the source styles for the Style Kit's CSS
  • /node_modules contains any packages required by this project's package.json file
  • /provisioning related to developing this project (i.e. relevant to developers rather than end-users)
  • /site contains the compiled or built version of the Jekyll end-user documentation, suitable for use as a document root
  • README.md main project README, containing instructions on how to create instances of this project (for development or projection) and only relevant to this audience
  • gulpfile.js main project task file, defining tasks such as Less style compilation and icon font class listing, relevant if customising the Style Kit yourself
  • All other files are related to developing this project (i.e. relevant to developers rather than end-users)

See the contributing section of this documentation for information on how to setup your own instance of the BAS Style Kit for development or customisation.
